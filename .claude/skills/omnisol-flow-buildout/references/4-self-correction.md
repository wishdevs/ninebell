# 단계 4 — 자가수정 루프

읽기/쓰기 프로브(단계 2·3)는 한 번에 안 맞는다. 이 루프는 무한 삽질을 막는 규율이다:
**재시도 상한 + 실패 증거 + 원인 분류 + 변경 이력.** 성공하면 실측값을 PROCESS.md 에 접고
❓→✅ 로 승격한다. 이 루프는 `omnisol-flow-prober` 서브에이전트가 수행하고, 메인에는
확정 결론만 회수한다.

## 루프 구조

```
for attempt in 1..MAX_ATTEMPTS_PER_STEP (기본 3):
    프로브 실행(헤드리스, 프로덕션 조건 재현: delay_scale·결의구분·params)
    스텝마다 스크린샷 + 그리드/DOM JSON 덤프 저장(backend/e2e/artifacts/)
    성공? → 실측값 확정 → PROCESS.md 패치 초안 → break
    실패? → 원인 분류 → 변경 1개만 적용(가설 1개씩) → 변경 이력 기록
if 상한 초과: 실패 증거 + 분류 + 시도 이력을 묶어 도메인 전문가(사용자)에게 에스컬레이션
```

한 시도에 **가설 하나만** 바꾼다(셀렉터 하나, 좌표 하나, 대기 방식 하나). 여러 개를 동시에
바꾸면 무엇이 고쳤는지 모른다.

## 원인 분류 (택1 이상)

프로브 실패를 반드시 아래로 분류한다 — 분류가 곧 다음 수정 방향이다:

| 분류 | 증상 | 1차 대응 (근거: erp-headless-grid-automation) |
|---|---|---|
| **셀렉터 드리프트** | 클래스/id/좌표가 안 잡힘 | `nbkit/omnisol/selectors.py` 단일소스 확인·갱신. DOM 덤프로 실제 id 확인 |
| **타이밍(time-scaling)** | 1.0 에선 되는데 delay_scale 에서 flake | 고정 settle → **조건 폴링**으로 전환. 프로브도 같은 scale 로 재현 |
| **필드 부재(문서종류 편차)** | 형제 복사 후 그 필드가 없음 | 라벨 존재 프로브 → **skip-if-absent**(`{ok, skipped}`), hard-fail 금지 |
| **트리거 미발화(setValue 오염)** | 값은 들어갔는데 저장 DB 에러/의존필드 빈칸 | setValue 금지 → 실 에디터 타이핑 + Tab 커밋 + 트리거 팝업 확인 |
| **숨은 백킹필드** | dataSource 엔 있는데 화면엔 안 보임 | `getDisplayValuesOfRow` 로 표시값 검증. 실제 위젯(부가선택/피커) 찾기 |
| **피커 미적용** | 적용/확인 눌러도 반영 0, 빈 행만 추가 | 결과행 **dblclick**. 부작용 빈 행은 ERP 방식으로 삭제 |
| **좌표(뷰포트)** | 픽셀 클릭이 빗나감 | 뷰포트(1600×1000) 고정. bbox 중심 재계산. `getCellRect` 신뢰 금지 |
| **권한/메뉴** | "메뉴 없음/권한 없음" 팝업 | 즉시 실패(MenuError). 사용자유형·딥링크·계정 권한 확인 |
| **데이터 없음** | 조회 결과 0건 | 업무 전제(기간·계정) 확인. 자동화 버그 아닐 수 있음 → 사용자 확인 |
| **팬텀 저장** | 저장 "성공"인데 재조회 시 0건 | 저장 전 모달/에디터 blur, 저장 후 재조회로 영속 검증(양성신호) |
| **코드/명칭 레벨 불일치** | 사용자가 준 코드가 실측 필드와 안 맞음 | 그 코드가 어느 레벨인지 실측 매핑(부서 `BG_CD` / 계정 `BGACCT_CD` / 피커행)해 사용자 확인. 예: 예산단위 "2005"가 계정코드 아닌 부서코드(회계팀)였음 — **사용자 제공 코드·명칭은 실측 매핑 전까지 가설** |

## 실패 증거 (반드시 남긴다)

- **스크린샷 every step** — 헤드리스는 사용자가 못 본다. 스텝별 PNG 로 스토리보드를 만들면
  도메인 전문가가 즉시 원인을 짚는다(과거 다회 조사를 이걸로 뚫음).
- **그리드/DOM JSON 덤프** — `getColumns()`(fieldName·visible·editor), `getJsonRows` vs
  `getDisplayValuesOfRow`, 열린 에디터의 모든 input(id·value·bbox).
- 아티팩트 경로는 `backend/e2e/artifacts/<probe>_<step>.png|json`. 회수 시 경로만 보고.

## 변경 이력 (PROCESS.md 검증 로그에 축적)

```
- <YYYY-MM-DD> attempt N (<probe.py>): 가설=<한 문장> / 결과=<성공·실패> /
  분류=<원인> / 증거=<artifact 경로> / 다음=<이어서 시도할 것>
```

## 에스컬레이션 규칙

**라이브 2~3회 실패하면 멈춘다.** 실 ERP 동작(어느 팝업이 뜨는지, 어떤 수동 제스처가 먹는지,
그 필드가 애초에 있는지)은 브루트포스보다 **손으로 하는 사람에게 묻는 게 빠르다.** 실패 증거
스토리보드 + 원인 분류 + 시도 이력을 묶어 메인 에이전트가 사용자에게 질문으로 올린다.
