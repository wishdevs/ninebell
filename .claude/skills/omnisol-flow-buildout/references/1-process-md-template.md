# 단계 1 — 플로우 명세 (PROCESS.md) 템플릿

**PROCESS.md 가 이 리포의 "에이전트별 flow.md"다.** 새 파일 형식을 만들지 말고 이 스켈레톤을
쓴다(실제 `trip_overseas/PROCESS.md`·`card_collect/PROCESS.md` 구조). 위치:
`backend/app/agents/<snake_name>/PROCESS.md`. 사람이 읽고 git 으로 버전관리하는 SOP.

## 작성 규칙

- **사용자 구술 그대로 먼저.** 화면/업무 스텝을 사용자가 말한 대로 적고, 기술 미확정 요소
  (셀렉터·좌표·필드id·컬럼·결의구분 value)는 `검증: ❓` 로 표시. 절대 추측을 사실처럼 쓰지 않는다.
- **업무 결정은 D1…Dn 으로 명시.** 값 규칙·기본값·파생 규칙(예: 회계일자=증빙일 최댓값)을 번호로.
  나중에 프로브가 뒤집으면 그 D 항목을 갱신.
- **비가역 액션 표시.** F7 저장, 코드피커 확정 '적용' 이후, F6/전표취소 삭제, 상신 → `⚠ 비가역`.
- **스텝마다 성공조건·실패조건.** "성공: 마스터 rowcount ≥ 1", "실패: '권한 없음' 팝업 → 즉시 중단".
- **❓ → ✅ 승격은 단계 2~4 프로브 이후에만.** 실측 로그를 `검증 로그` 에 남기고 값을 접는다.
- `RESOLUTIONS.md` 표에 새 행 등록, 형제 PROCESS.md 를 `> 형제:` 로 크로스링크.

> 상태: **v0 — <근거: 신규/형제 복사>. 헤드리스 라이브 프로브 <미실행/완료>.**

## 개요
<입력 방식(실행 전 폼 vs HITL 개입), 행 구조(단건/다건 F3 반복), 완주 조건(폼값 채움→F7)>

## 확정된 업무 결정(사용자, <YYYY-MM-DD>)
- **D1 입력 방식**: 실행 전 폼(개입 없음) | HITL 그리드 편집. params 네임스페이스 `<ns>`.
- **D2 행 구조**: 결의서 1장에 N건, F3 반복 | 단건.
- **D3 결의구분**: **"<라벨>"**(value <NN> — 검증: ❓ 라이브 확인).
- **D4 회계일자(ACTG_DT)**: <파생 규칙>. compact 'YYYYMMDD'.
- **D5 증빙유형**: 코드 **<NN> "<명칭>"**.
- **D6 거래처**: <규칙>.
- **D7 적요**: <자유입력 | 유형별 기본문구>.
- **D8 예산단위**: **"<명칭>"** — 부서 × cost_type(판/제) 정확매칭. 검증: ❓.
- **D9 (세금)계산서일**: detail 셀 **`START_DT`** = 행별 증빙일.
- **D10 금액**: <공급가액 총액 입력 | 계산 규칙>. 필드 검증: ❓.
- **D11 프로젝트**: 피커/카탈로그, WBS_NO 정확매칭. 기본값 규칙.
- **D12 상대계정거래처**: <규칙 | 해당 없음(문서종류 편차)>.
- **D13 저장**: F7 저장까지 자동. ⚠ 비가역. 상신은 사용자가 ERP 에서 직접.
