# 단계 0 — 대상 조사 / 실현가능성 게이트

목적: **코드·명세를 쓰기 전에** 이 자동화가 가능한지, 안전하게 할 수 있는지 판단한다.
산출물은 한 줄짜리 **진행/중단 판단** + 진입 경로 메모. 애매하면 진행하지 말고 사용자에게 확인.

## 필수 입력 게이트 — 요청에 없으면 **먼저 묻는다** (추측 금지)

요청 프롬프트가 아래 3가지를 담고 있지 않으면, 단계 1로 넘어가기 전에 **사용자에게 먼저 묻는다.**
답을 받기 전에는 진입 경로·사용자유형·기본 플로우를 "확정"으로 적지 않는다(전부 `검증: ❓` 유지).

1. **대상 페이지/화면** — 어느 옴니솔 메뉴·화면인가?(메뉴명·딥링크, 예 `/FI/GLDDOC00300`.)
   그리고 그 화면이 **회계(FI)인지 인사(IM)인지** — 사용자유형이 갈린다. ⚠ 새 문서종류가 기존
   에이전트(출장·카드)와 **같은 화면·사용자유형이라고 가정하지 말 것.** `RESOLUTIONS.md` 의 "예정"
   행은 계획일 뿐, 실제 처리 경로의 증거가 아니다(예: 경조금이 회계 결의서가 아니라 인사 복리후생
   폼일 수 있다 — 확인 전까지 최상위 ❓).
2. **계정** — 어떤 계정으로 로그인하나? 쓰기 검증(단계 3/7)용 **격리된 테스트 계정**은 무엇인가?
3. **기본 플로우** — 사람이 이 업무를 손으로 하는 순서는?(진입 → 필드 입력 → 증빙 → 저장까지
   스텝별로.) 이게 단계 1 PROCESS.md 의 뼈대가 된다. 모르면 시연·설명을 요청한다.

셋 중 하나라도 비면 그게 곧 STOP 사유다 — 추측으로 메우지 말고 **질문으로 채운다.**

## 게이트 체크리스트

| 항목 | 확인 내용 | 옴니솔에서 보통 |
|---|---|---|
| 로그인 방식 | id/pw? SSO? 세션 만료? | id/pw, 매 실행 1회 로그인 후 즉시 폐기(비저장) |
| 2FA / OTP | 추가 인증 단계가 있나 | 사내 ERP — 대개 없음. 있으면 **중단**하고 사용자와 협의 |
| 봇 탐지 | rate-limit·행동분석·디바이스 지문 | 사내망 — 대개 없음. 단 과도한 반복은 서버 부하 주의 |
| CAPTCHA | 로그인/저장 시 캡차 | 없음. 생기면 **중단** |
| robots / ToS | 자동화 허용 범위 | 사내 승인 업무. 그래도 파괴적 액션은 사용자 승인 필수 |
| 테스트 계정 | **격리된 계정**이 있나(실운영 계정 아님) | 필수. 쓰기 검증(단계 3)은 테스트 계정에서만 |
| 진입 경로 | 사용자유형(인사/회계)·메뉴 딥링크·결의구분 | 아래 "진입 경로 확정" |

## 진입 경로 확정 (옴니솔 특이사항)

새 화면/문서종류는 **진입 앞단**이 결정적이다. `nbkit/OMNISOL_NOTES.md` §4·§6·§9 근거:

1. **사용자유형(인사/회계)** — 접근 모듈이 갈린다(인사→IM 재고, 회계→FI 재무회계).
   전환은 **좌표 실클릭 → 변경적용**(JS `.click()`/Kendo `.value()` 는 핸들러를 못 깨운다).
   결의서입력(GLDDOC00300)은 **회계**.
2. **메뉴 딥링크 우선** — `/FI/GLDDOC00300` 형태. 폴백(사이드바 플라이아웃)은 라이브 전용.
   진입 판정은 `.dews-ui-grid` 개수. "메뉴 없음/권한 없음" 팝업이면 90초 헛돌지 말고 즉시 실패.
3. **결의구분(ABDOCU_FG_CD) 값** — 문서종류를 가르는 코드(예: 출장 국내 53, 해외 54).
   신규 문서종류의 결의구분 라벨·value 는 **단계 2 프로브로 실측 확인**(여기선 후보만 기록).

`common/nodes.py` 의 진입 앞단(login→user_type→menu_nav→set_gubun→add_row→open_evdn)은 이미
공유 자산이다 — 새 문서종류는 이걸 **재사용**하고 결의구분/증빙유형/후속 폼만 갈아끼운다.

## 중단(STOP) 조건

하나라도 해당하면 진행하지 말고 사용자에게 보고:

- 2FA/CAPTCHA 가 걸려 있다(헤드리스로 안정 통과 불가).
- 격리된 **테스트 계정이 없다**(쓰기 검증을 실운영 계정에서 하면 안 됨).
- 진입 경로(메뉴·사용자유형·결의구분)가 불명확하고 도메인 전문가 확인이 안 된다.
- 대상 액션이 되돌릴 수 없고(상신·확정 등) 삭제 경로가 없다.

## 통과 시 산출물

```
## 실현가능성 판단 — <문서종류>
- 진행/중단: 진행
- 로그인: id/pw(비저장), 2FA 없음, CAPTCHA 없음
- 테스트 계정: <계정> (격리 확인)
- 사용자유형: 회계
- 메뉴 딥링크: /FI/GLDDOC00300
- 결의구분 후보: "<라벨>" value ❓(단계 2 실측)
- 비고: <주의사항>
```

이 판단을 PROCESS.md 초안(단계 1) 상단 상태 줄에 반영한다.
