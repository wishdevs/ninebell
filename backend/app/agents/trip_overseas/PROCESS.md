# 출장(해외/정산서) 결의서입력 — 프로세스 정의 (trip-overseas)

> 결의서입력(GLDDOC00300) 문서 종류 중 **출장(해외·정산서)**. 패밀리 인덱스: [../RESOLUTIONS.md](../RESOLUTIONS.md).
> 상태: **v0 — 국내/자차(trip-domestic) 복사·수정. 헤드리스 라이브 프로브 미실행(ERP 필드 가정 존재).**
> 형제(기본틀 동일): [../trip_domestic/PROCESS.md](../trip_domestic/PROCESS.md).

## 개요

국내/자차(trip-domestic)와 **기본틀 동일** — 모든 입력이 사용자 제공이라 **실행 전 폼**으로 params
를 받고, 그래프는 HITL 없이 무개입 완주(폼값 채움 → F7 저장)한다. 결의서 1장에 여러 정산 행을
F3 반복으로 채운다. 국내와 달리 **유형(통행료/유류비) 구분이 없다** — 모든 행이 동일 형태다.

## 확정된 업무 결정(사용자, 2026-07-07)

- **D1 입력 방식**: 실행 전 폼(개입 없음). params 네임스페이스 `trip`.
- **D2 여러 행**: 결의서 1장에 정산 행 N건, F3 반복.
- **D3 결의구분**: **"출장(해외·정산서)"**(가운뎃점 `·`, value 54 추정 — ❓ 라이브 확인).
- **D4 회계일자(마스터 ACTG_DT)**: **계산서일(증빙일) 중 가장 마지막일**로 파생(사용자 확정, 국내
  동일). compact 'YYYYMMDD' 세팅. 별도 '출장일' 입력 없음.
- **D5 증빙유형**: 코드 **10 "규정에의한 비용정산"**(국내 동일).
- **D6 거래처**: **작성자 본인 이름 런타임 검색**(전 행 동일, 유형 구분 없음).
- **D7 적요**: **자유 입력**(행별). 예: "해외출장 일비", "해외출장 경비".
- **D8 예산단위**: **"여비교통비-해외출장" 고정** — 부서(BG_NM) × cost_type(판/제) 조합 정확매칭
  (국내의 "국내출장"만 "해외출장"으로 교체). ❓ BGACCT_NM 정확 명칭·행수 라이브 확인 필요.
- **D9 (세금)계산서일**: detail 셀 **`START_DT`** = 행별 증빙일(국내 동일).
- **D10 공급가액**: **총 금액 입력**. 현재 `set_transaction_amount`(SPPRC_AMT2+공급가액+합계 동일
  세팅) 재사용. ❓ 해외 정산서의 금액 필드가 국내와 동일한지 라이브 확인 필요(가정).
- **D11 프로젝트**: 국내 출장과 동일(피커/카탈로그, WBS_NO 정확매칭). 기본값=팀 비용구분 프로젝트
  (제조원가→500/판관비→800, `/me/trip-defaults`).
- **D12 상대계정거래처**: **작성자 본인**(BFC_PARTNER_CD setValue, 국내 동일 — 사용자 확정).
- **D13 저장**: F7 저장까지 자동, 상신은 사용자가 ERP 에서 직접.

## ⚠ 저장·상신 주의(노티스 — 폼에 표시)

해외출장 **일비가 두 달에 걸치면**(예: 3/30~4/6) **회계일자별로 별도 결의서로 나눠 상신**한다:
- 회계일자 **3/31**로 설정한 일비(3/30~3/31분)
- 회계일자 **4/6 이후자**로 설정한 일비(4/1~4/6분)

우리 그리드는 F3로 새 행을 만드는 구조이므로, **월이 바뀌는 건은 각각 별도 실행(run)** 으로 처리
한다(회계일자는 폼의 계산서일 최댓값으로 자동 지정되므로, 한 run 의 행들은 같은 달로 유지). 폼
상단 노티스로 안내한다.
